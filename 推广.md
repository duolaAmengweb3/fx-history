做跨境电商久了才发现,每年被汇率悄悄"偷走"的钱,比想象中多得多。

Amazon 美欧日三站多币种到账,PayPal / Shopify / Stripe 结算,Wise / Payoneer 提现——每个月对账都在切换 5-6 种汇率,手算一次要 2 小时,还不一定算对。

关键是,平台给的结算汇率,总比外面市场中间价低那么一点。你隐约知道,但说不清楚差多少,也没工具能让你实时算清楚。

上个月对账,我发现 Amazon 本月结算比 ECB 中间价低了 1.2%。$18,000 少到手 ¥1,500 多。这数字让我愣了一下——平台悄悄扒了我一笔,而我居然没有工具能实时看清楚。

查了一圈市面上的工具:
- XE、OANDA 是英文通用,不懂跨境卖家场景
- 马帮、易仓是跨境 ERP,按月收费 ¥300 - ¥3,600
- 彭博、Reuters 是金融级工具,一年几万美元
- Amazon Currency Converter 不告诉你扒了你多少

给跨境卖家的、懂对账场景的、永久免费的汇率工具箱,这事儿居然没人做。

9 个工具,分 3 组。花了两周时间,基于欧洲央行(ECB)官方汇率数据做的,永久免费。

⸻

👇 日常换算(3 个)

1)单次换算
任意日期 + 任意币种对。周末节假日 ECB 不发布汇率,很多工具给错或白屏,我们自动回退到最近交易日,UI 上明确提示"原日期为非交易日,已回退到 2026-04-17"。

2)一币对多币
1 个源币同时换 5 个目标币。给多站定价、分析多平台收入结构时,一屏看清不用反复切换。

3)反向定价
输入期望到手 ¥3,000,系统反推 Amazon 应该定多少美元。别再拿计算器手算了。

⸻

🔥 平台分析(3 个,其他免费工具不做的)

4)平台结算
12 个平台预设(Amazon 美欧英日、Shopify Payments、Stripe、PayPal、Wise、Payoneer、TikTok Shop 美英、Etsy),每个都按该平台真实结算周期自动取 ECB 汇率:
- Amazon:14 天周期均价
- Shopify Payments / Stripe:T+2 每日
- PayPal:即时
- Wise:市场价 + 透明费率

输入结算金额和日期,系统输出:ECB 中间价理论到账 vs 平台实际到账,看清被扒多少,并对比 Wise / Payoneer 能省多少。

举例:$50,000 走 Amazon 美国,ECB 理论 ¥341,802,Amazon 实到 ¥337,957,差 ¥3,845(1.13%)。改用 Wise(0.5%)能省 ¥2,136。

5)结算窗口
"下次 Amazon 结算还有 5 天,但美元今天涨了 1%——我该手动提现还是等平台?"

系统分析本结算周期内汇率高低点、当前所处分位,给出明确建议:"✅ 当前汇率高于周期均值 1.5%,适合立刻结汇,按 $50k 能多拿 ¥3,842"——不玄学,不靠感觉。

6)毛利汇率敏感度
输入采购成本(RMB)+ 售价(外币)+ FBA / 运费 + 平台佣金,系统算:
- 当前单件毛利和毛利率
- 过去 1 / 3 / 5 年毛利随汇率波动曲线
- 盈亏平衡汇率(汇率跌到这个数,这个 SKU 开始亏损)
- 7 档敏感度表(6.0 / 6.2 / 6.5 / 6.8 / 7.0 / 7.2 / 7.5)各自毛利

如果定价结构本身在亏,直接提示"结构性亏损,汇率再涨也救不回来,需要提价或换品类"。

⸻

📊 批量 / 报表(3 个)

7)CSV 批量换算
上传对账表(date,amount,from_currency),按每行日期和币种批量换算为目标币。周末自动回退,结果导出 CSV。Amazon / Shopify 月度几百行对账单,10 秒搞定。

8)月度对账报表
选月份 + 多平台多币种收入,输出:
- 月初 / 月均 / 月末 三种汇率口径并列展示
- 每个来源可选平台预设,自动扣减平台汇差
- 可选上月对比,直接看出本月比上月多/少多少
- 一键打印 PDF 给会计

以前手动做这张表至少 2 小时,现在 10 分钟。

9)历史趋势
任意货币对 1 / 3 / 5 / 10 年长期曲线 + 当前历史分位。

"USD/CNY 当前 6.82,位于过去 3 年的 92 分位(偏高)"——决定锁汇时机有数据依据,而不是凭感觉。

💼 实际怎么用(5 个跨境卖家日常场景)

场景一:月末关账(每月必做)
打开"月度报表" → 填入 Amazon 美 $8,500、Amazon 欧 €3,200、TikTok Shop 英 £2,100 → 为每个来源选对应平台预设 → 系统自动算出人民币合计 + 平台汇差扣减 + 上月同比涨跌 → 一键打印 PDF 发给会计。
原来 2 小时的活,10 分钟搞定。

场景二:新品定价
工厂报完采购成本 → 打开"毛利敏感度" → 输入成本 / 目标售价 / FBA / 佣金 → 看敏感度表:当前毛利率多少,如果美元跌到 6.5 会变多少,跌到 6.0 会不会亏 → 定价时在当前汇率基础上预留 5% 缓冲。
避免汇率一波动就出现隐形亏损 SKU。

场景三:锁汇决策
手里一笔美元应收账款 → 打开"历史趋势"选 USD/CNY + 3 年 → 看当前分位(高于 85 分位说明偏高,可锁;低于 30 分位可以等) → 再打开"结算窗口"看本周期的位置 → 综合判断现在结汇还是等。
告别"感觉该结汇"。

场景四:开新平台前的收款渠道选择
要接入 Shopify Payments,但不确定 vs Wise vs Payoneer 哪个更划算 → 打开"平台结算"分别选三个预设 → 看 10 万美元月营收在不同平台的年度汇差差异 → 差异可能有几万人民币 → 用数据决定接哪个渠道,不拍脑袋。

场景五:年度预算规划
老板要按人民币定 2026 年 GMV 目标 → 打开"历史趋势"看 USD/CNY 过去 3 年均值(比如 7.05) → 汇报时用更保守的值(比如 6.8)作为预算基准 → 给自己留汇率波动空间,老板看得懂,汇率真的波动时不会被追责。
做预算有底气。

📡 技术 + 数据边界(诚实说)

数据源:欧洲央行(ECB)官方每日参考汇率,via Frankfurter API
覆盖:30 种主流货币,1999-01-04 至今
技术栈:Next.js 16 + React 19 + TypeScript,纯前端无后端
隐私:你的 CSV / 金额 / SKU 数据全部在浏览器处理,不上传任何服务器
成本:永久免费 · 无广告 · 无账号 · 无订阅

几个边界坦白说:
- ECB 只发 30 种货币,不包含 RUB / TWD / VND / AED / SAR 等。做俄罗斯 / 中东 / 台湾 / 越南市场需要结合其他数据源
- CNY 是在岸人民币,不是离岸 CNH。两者差异通常 0.1-0.5%,做管理报表够用
- 本工具用于内部管理报表和决策参考,不能作税务凭证(中国境内税务应以中国人民银行公布的中间价为准)
- 平台汇差数值基于平台公开文档 + 行业实测的参考区间,实际以你账户收到为准
