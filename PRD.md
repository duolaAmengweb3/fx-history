# FXHistory · 跨境卖家汇率工具 · 产品需求文档

> **版本**: v1.0(MVP 定义) · **编写日期**: 2026-04-19
> **定位**: 免费 · 纯前端 · 跨境电商卖家专用汇率换算 / 核算 / 风险参考工具
> **项目代号**: FXHistory
> **数据源**: [Frankfurter API](https://api.frankfurter.dev/v1/)(ECB 官方数据)

---

## 一、产品定位

### 一句话定位

**给跨境电商卖家的"汇率瑞士军刀"** —— 不仅查汇率,还能**结合平台结算周期**做多币种到账核算、历史月度回溯、利润波动分析、锁汇决策参考。

### 核心价值

通用汇率工具(Google / XE / 小工具)只解决"1 美元 = 多少人民币"。但跨境卖家的真实问题是:

| 卖家真实问题 | 通用工具 | FXHistory |
|---|---|---|
| "Amazon 上月 14 天结算周期的均价汇率是多少" | ❌ | ✅ 预设结算周期一键算 |
| "我三个平台(美 / 欧 / 日)这个月收入合并换算人民币是多少" | ❌ | ✅ 多币种合并报表 |
| "Amazon 实际给我的汇率比 ECB 低了多少" | ❌ | ✅ 独家对比 |
| "现在 USD/CNY 是处于过去 3 年的高位还是低位?该不该锁汇" | ❌ | ✅ 历史分位看板 |
| "给我出一张月度汇率对账表用于财务做账" | ❌ | ✅ PDF/CSV 导出 |

### 用户画像

- **主要用户**:中国大陆跨境电商卖家(Amazon / Shopify / eBay / TikTok Shop / Etsy / 独立站)
- **规模**:中小卖家(月 GMV $1k - $100k),大卖有 ERP 不需要
- **角色**:老板 / 运营 / 财务 / 会计
- **使用场景**:
  1. 月末核账(主要场景,占 60%)
  2. 报价定价(给新产品定售价,折算采购成本毛利)
  3. 决策锁汇时机(偶尔但重要)
  4. 跨平台收入合并汇总(跨月 / 跨季度)

### 非目标用户(不做的场景)

- 企业级财务系统(不做 SSO、不做 ERP 对接、不做审批流)
- 外汇交易员 / 金融机构(我们提供的是 ECB 参考中间价,不是实时交易报价)
- 合规税务凭证(中国税务局认可的是人民银行中间价,**我们只能作为对照参考,不能作为法定凭证**,这点必须在产品内标注)

## 二、数据源能力边界(必读)

### ✅ 可用

- **30 种货币**:AUD / BRL / CAD / CHF / **CNY** / CZK / DKK / **EUR** / **GBP** / **HKD** / HUF / **IDR** / ILS / **INR** / ISK / **JPY** / **KRW** / **MXN** / **MYR** / NOK / NZD / **PHP** / PLN / RON / SEK / **SGD** / **THB** / TRY / **USD** / ZAR
- **时间范围**:1999-01-04 至今(覆盖绝大多数财务场景)
- **更新频率**:每工作日欧洲中部时间 16:00 左右(ECB 发布后)
- **API 无 key、无配额上限、CORS 开放**(`*`)
- **CDN 级缓存**(Cache-Control: 24h),客户端重复请求不占用额度

### ❌ 不支持

- **缺货币**:RUB(俄)、AED/SAR(中东)、VND(越南)、TWD(台湾)、NGN(尼日利亚)、EGP(埃及)
  - **影响**:做中东 / 东欧 / 越南市场的卖家会有盲点
  - **应对**:在币种选择器明确标注"基于 ECB 数据,暂不支持 X Y Z",并引导到官方说明页
- **缺日期**:周末(周六周日)和 ECB 节假日(圣诞、复活节等)
  - **应对**:自动 fallback 到最近交易日,并在 UI 显示"使用 YYYY-MM-DD 的汇率(原日期为非交易日)"
- **非中间价**:ECB 参考汇率是**外汇市场中间价**,不等于银行柜台价 / 平台结算价 / 实盘交易价,用户看到的"实际到账"会有 0.5%-2% 差。**产品要明确这一点,否则误导**

## 三、功能清单

> **MVP 标记**:🟢 必做(首发)/ 🟡 重要(v1.1)/ ⚪ 可选(v2+)

### 🏠 模块 A:核心换算

- **A1. 单次换算器** 🟢
  - 输入:金额 + 源币种 + 目标币种(1 个或多个)+ 日期(默认今日)
  - 输出:该日汇率 + 换算结果
  - 交互细节:日期选择器自动禁用未来日期 + 小于 1999-01-04 的日期;选到非交易日自动提示"将使用 YYYY-MM-DD"
  - 键盘快捷:Enter 换算,Cmd+K 聚焦输入框
- **A2. 一币兑多币** 🟢
  - 一次输入 $100,同时输出对 CNY / EUR / GBP / JPY / HKD 的换算
  - 目标币种列表可在"设置"中自定义预设组
- **A3. 反向换算** 🟢
  - "我想给美国客户报价,希望人民币收入是 ¥3000,该定多少美元"——反推

### 🏠 模块 B:平台结算周期预设(差异化核心 1)

- **B1. 平台预设库** 🟢
  - 预置周期模板:
    - **Amazon 美国**: 每 14 天结算一次,按结算日前 N 天均价
    - **Amazon 欧洲 / 日本**: 同上但币种不同
    - **Shopify Payments**: T+2 日结算(国家依赖)
    - **Stripe**: T+2 ~ T+7(国家依赖)
    - **PayPal**: T+1 或手动提现
    - **Wise Business**: 实时市场价 - 0.35% 服务费
    - **Payoneer**: 手动提现,汇率由 Payoneer 决定(市场价 - ~2%)
  - 数据结构在 [`TECH.md#platform-presets`](TECH.md)
- **B2. 平台汇率差计算器** 🟢(本产品**最强差异化**)
  - 用户输入:平台名 + 到账金额 + 结算日期
  - 系统输出:
    - ECB 官方中间价下应该到账 ¥X
    - 平台实际到账 ¥Y
    - **差额 = ¥X - ¥Y(= 平台赚的汇差 +/- 手续费)**
    - 换算汇率差百分比
  - 让卖家第一次看清楚"Amazon Currency Converter for Sellers 吃了我 1.2%" 这种数字
- **B3. 最佳结算窗口提示器** 🟡
  - 给定结算周期(例如 "Amazon 下一次结算日是 5 月 6 日,回看 14 天")
  - 输出这 14 天内每天汇率曲线 + 最高 / 最低 / 平均
  - 帮卖家判断"是不是该手动提前提现"

### 🏠 模块 C:批量换算与 CSV(差异化核心 2)

- **C1. CSV 批量上传** 🟢
  - 支持字段:`date, amount, from_currency, [to_currency]`
  - 示例模板下载
  - 支持任意行数(前端用 Web Worker 处理,避免卡 UI)
  - 自动识别常见导出格式(Amazon / Shopify / PayPal 月度对账单)—— v1.1 做"智能识别"
  - 导出:原表 + 新增"目标币金额"列
- **C2. 月度对账报表生成器** 🟢
  - 输入:时间范围(如 2026-03 整月) + 多个币种收入总额
  - 输出 HTML / PDF 报表,含:
    - 各币种总收入
    - 月初 / 月末 / 月内平均 3 种汇率下的人民币值
    - 按日汇率逐日累加的**精确值**(财务本位)
    - 汇率波动对总收入的影响(如 +¥1,200)
- **C3. 多平台合并报表** 🟡
  - 分别导入 Amazon 美欧日 + Shopify + TikTok Shop 月度对账
  - 一键合并成"本月总收入 ¥X,分币种贡献:美 $Y → ¥A,欧 €Z → ¥B..."
  - 自动展示"汇率波动导致本月实际入账较上月多了 / 少了多少"

### 🏠 模块 D:历史趋势与风险参考

- **D1. 汇率曲线图** 🟢
  - 单币对 1 年 / 3 年 / 5 年 / 自定义区间历史曲线
  - 支持对数坐标、MA7 / MA30 移动平均线
  - 鼠标悬停显示精确值
  - 节假日空心点处理(不做直线插值,显示为虚线)
- **D2. 多币对叠加对比** 🟢
  - 同时查看 USD/CNY、EUR/CNY、GBP/CNY 走势
  - 归一化对比(都以 100 为基准)
- **D3. 历史分位仪表盘** 🟡(差异化核心 3)
  - 选一个货币对,显示:
    - 当前汇率 vs 过去 1 年 / 3 年 / 5 年的分位数
    - 如 "USD/CNY 当前位于过去 3 年的 92 分位(偏高),适合结汇,不适合购汇"
    - 不做预测,只做历史参考(避免金融合规风险)
- **D4. 关键事件时间线叠加** ⚪(v2)
  - 在图表上标注重大事件:2015 汇改、2020 疫情、2022 美联储加息周期等
  - 帮助用户理解波动来源

### 🏠 模块 E:毛利与风险分析

- **E1. 毛利汇率敏感度分析** 🟡
  - 输入:采购价(RMB)+ 售价(外币)+ FBA/运费(外币)+ Amazon 佣金比例(默认 15%)
  - 输出:
    - 按当前汇率的毛利率
    - 过去 12 个月毛利率波动图(若汇率从 7.3 跌到 6.9,毛利会变 X%)
    - "汇率跌到 X 时,毛利为 0(亏损线)"
- **E2. 批量 SKU 毛利测算** ⚪(v2)
  - CSV 上传 SKU 清单,批量算毛利
- **E3. 锁汇决策参考面板** ⚪(v2)
  - 不做金融建议,只做数据呈现:显示历史 12 个月波动区间、年化波动率、如果 1 个月前锁汇今天会怎样

### 🏠 模块 F:工具 / 分享 / 订阅

- **F1. URL 参数化分享** 🟢
  - 任意查询结果生成可分享链接(`/?from=USD&to=CNY&amount=100&date=2026-03-15`)
  - 方便卖家在微信群丢链接给同事
- **F2. 查询历史本地记忆** 🟢
  - localStorage 存最近 20 次查询
  - 右上角"历史"Drawer 一键回溯
- **F3. 收藏夹 / 常用货币对** 🟢
  - 固定自己最常查的 3-5 个币对到首页顶栏
- **F4. 中英双语切换** 🟢
  - 默认中文;URL `?lang=en` 切换,保留产品海外复用空间
- **F5. 浏览器扩展** ⚪(v2)
  - Chrome 扩展:选中数字 + 右键 → "换算成人民币"
- **F6. 每日汇率邮件订阅** ⚪(v2,需薄后端)
  - 需要接入邮件服务,首版纯前端不做

### 🏠 模块 G:杂项

- **G1. 帮助中心** 🟢
  - 数据来源说明
  - 为什么没有 RUB / AED 等(引导理解)
  - 免责声明(非税务凭证、非金融建议)
- **G2. Changelog** 🟢
  - 功能迭代透明化
- **G3. 反馈入口** 🟢
  - 一个邮箱链接 + GitHub Issue 链接即可,不做站内反馈系统

## 五、关键用户旅程(User Journey)

### 旅程 1:月末核账的张老板(最高频)

1. 月末关账,张老板打开 FXHistory
2. 首页直接看到"📊 月度对账工具" 大按钮,点击进入 C2 模块
3. 选择月份 `2026-03`
4. 输入或粘贴三行数据:
   - Amazon 美国 $8,500
   - Amazon 欧洲 €3,200
   - TikTok Shop 英国 £2,100
5. 选换算目标 CNY,点击"生成报表"
6. 系统输出:
   - 按 3 月平均汇率:¥86,340
   - 按 3 月末汇率:¥87,020
   - 按逐日汇率精确累加(若有逐日收入明细):¥86,680
   - **一键导出 PDF**,交给会计

### 旅程 2:决策锁汇的李老板(低频高价值)

1. 听同行说"美元要跌了",想看看现在到底什么位置
2. 打开 FXHistory → D1 + D3
3. 选 USD/CNY → 看到:
   - 当前 7.18
   - 过去 3 年中位数 7.05,当前位于 78 分位
   - 过去 1 年最高 7.32,最低 6.98
4. 判断"现在偏高,下月回款可以收到美金就结汇"

### 旅程 3:新品定价的王运营

1. 工厂报价 ¥85,要定 Amazon 美国售价
2. 打开 FXHistory A3(反向换算)
3. 按毛利模型倒推,定价 $19.99
4. 进入 E1 看"若美元跌到 6.8,毛利还剩多少" → 得到风险区间
5. 最终定价 $21.99 为保险

### 旅程 4:临时查一个数的小白

1. 随手搜"美元 人民币 2025年3月"
2. 打开 FXHistory 首页
3. 直接输入 100 USD → 选 2025-03-01 → 立刻看到结果 ¥X
4. 旁边附带"月平均 ¥Y / 月末 ¥Z",顺手得到 bonus 信息

## 八、风险和限制

### 数据风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| Frankfurter API 停服 | 产品直接不能用 | 准备 fallback 到 exchangerate.host 或 ECB 原始数据;每日自动镜像到 Vercel KV |
| ECB 数据延迟 | 当日汇率当晚才有 | 首页显示"数据截至 YYYY-MM-DD,欧洲时间 16:00 后更新" |
| 缺货币被需求 | 用户抱怨 | 明确说明 + 反馈收集,需求量大时人工补充 CNH(离岸人民币,用其他数据源补) |

### 合规风险

- **绝不声称可用于法定税务** —— 页面所有地方有"参考对照,非税务凭证"角标
- **绝不做金融建议 / 预测** —— 分位数仅为历史事实描述,不说"会涨 / 会跌"
- **ECB 数据许可**:公共领域,自由使用(CC-BY 归因即可);项目 footer 挂 "Exchange rates by European Central Bank via frankfurter.dev"

### 技术风险

| 风险 | 缓解 |
|---|---|
| CDN 缓存失效突增请求 | 客户端再加 localStorage 二级缓存,同一查询 24h 不重复请求 |
| 大 CSV 卡死浏览器 | Web Worker 处理,分批次调用;10 万行以上给出"建议拆分"提示 |
| PDF 导出中文字体过大 | 使用 jsPDF + 按需加载思源宋体子集 |

## 十、附录:数据结构草案(供技术文档细化)

### 平台预设(摘录)

```ts
interface PlatformPreset {
  id: string;           // "amazon-us"
  name: string;         // "Amazon 美国"
  platform: "amazon" | "shopify" | "paypal" | "stripe" | "wise" | "payoneer";
  settlementCurrency: string;  // "USD"
  settlementCycleDays: number; // 14
  settlementMethod: "average" | "closing" | "daily";
  platformFxMarkup?: number;   // 0.01 表示平台在中间价上加 1%
  notes?: string;              // 说明文字
}
```

### 月度报表输出

```ts
interface MonthlyReport {
  month: string;            // "2026-03"
  targetCurrency: string;   // "CNY"
  items: {
    source: string;         // "Amazon 美国"
    amount: number;
    currency: string;
    converted: {
      byAvgRate: number;
      byClosingRate: number;
      byDailyActual: number | null;
    };
  }[];
  totals: {
    byAvgRate: number;
    byClosingRate: number;
    byDailyActual: number | null;
  };
  rateTable: { date: string; [currency: string]: number | string }[];
  metadata: {
    dataSource: "Frankfurter / ECB";
    generatedAt: string;
    disclaimer: string;
  };
}
```
